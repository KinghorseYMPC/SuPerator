# external-research-review

## Purpose

Define how to review external automated research repositories or lists without unsafe execution or direct copying.

## When to use

- Reviewing an external research Agent, skill collection, plugin collection, survey, or awesome list.
- Deciding whether an external idea should influence local generic workflow skills.
- Checking whether a source is safe and legally suitable for adaptation.

## Procedure

1. Record source name, URL, access date, topic, and reviewer.
2. Check the license before adapting ideas.
3. Read only explanatory files such as README, LICENSE, docs, and skill indexes.
4. Do not execute scripts, run Agents, install dependencies, or vendor the repository.
5. Extract general workflow ideas instead of copying source text.
6. Reject or defer content that is license-unclear, too domain-specific, execution-dependent, or strategy-like.
7. Rewrite accepted ideas as local, generic procedures.
8. Update the external intake log, skill index, registry, and tests.

## Guardrails

- Treat external repositories as untrusted during intake.
- Do not copy large verbatim passages.
- Do not import task-specific execution routes, model choices, data plans, or score optimization advice.
- Do not commit downloaded repositories or large external artifacts.
- If network access fails, record the failure and proceed only from already available local context.
